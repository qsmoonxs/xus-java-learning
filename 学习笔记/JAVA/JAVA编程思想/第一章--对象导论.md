### 用引用操纵对象
- 尽管一切都看作对象，但操纵的标识符实际是对象的一个引用
``` String s;``` 这里创建的只是引用不是对象，引用可以单独存在

----------


### 存储到什么地方
- 寄存器 
- 堆栈 RAM中 堆栈指针向下移动分配新内存，向上移动释放内存，JAVA系统必须知道存储在堆栈中所有项的确切生命周期以便释放和分配，虽然有些JAVA数据存储在堆栈中--特别是对象引用，但是JAVA对象并不存储在其中，基本类型（方法的本地变量）存储在此，如果是类的字段那还是跟对象一起在堆里
- 堆（内存池）RAM中 存放所有JAVA对象，编译器不需要知道存贮的数据在堆里的存活时间 new String（）时就在堆里分配内存，创建一个数组对象时实际是创建一个引用数组（堆栈指针只移动一格），并且每个引用对象（在堆中）都初始化为null
- 常量存储 常量值通常直接放在程序代码内部
- 非RAM存储 数据完全存活于程序之外，不运行时也可以存在 比如持久化对象

### 创建基本类型的数组时候 是存放到堆还是堆栈？？？？ 其实跟数组对象时一样的 只是初始化的值不一样 
基本类型的数组创建的其实还是引用数组，在堆栈中并且初始化为0，但是真实的数据存放在堆中，比如 ```int[] a = new int[100]``` 数组对象在堆中，连续开辟100个空间，每个数据存放其中并和相应的 在堆栈中的 引用想关联 基本类型数组必须指定大小

### 什么时候销毁对象
- 基本类型数据在作用域结束的时候销毁，JAVA中作用域就{}
- JAVA对象不具备和基本类型一样的生命周期，new一个对象时 他可以存活于作用域之外 但是这个对象引用在作用域终点消失
,java的垃圾回收器会监视对象辨别哪些不需要再用进而销毁

### 类
- 类的字段是基本类型的话 会初始化值，在方法里的话不会
- JAVA的方法决定了一个对象能接受什么样的消息，调用方法的行为被称为发送消息给对象
- 每个类都有class对象，所以每个类本身都是在堆中有存储空间的，所以当有静态字段时 他可以调用
